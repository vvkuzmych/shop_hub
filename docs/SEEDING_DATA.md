# Database Seeding Guide

## Overview

ShopHub provides multiple seed files to populate your database with sample data for development and testing. This guide explains how to use each seed file and what data they create.

## Available Seed Files

### 1. Main Seed File (`db/seeds.rb`)

The main seed file creates a complete e-commerce dataset including:

- **Users**: Admin and customer accounts
- **Categories**: Hierarchical product categories (Electronics, Clothing, etc.)
- **Products**: Sample products with prices, stock, and SKUs
- **Orders**: Sample orders in various statuses
- **Addresses**: Shipping and home addresses for users
- **Attachments**: Product images and manuals
- **Comments**: Product and order comments
- **Reviews**: Product reviews (if applicable)

#### Usage:

```bash
cd /Users/vkuzm/RubymineProjects/shop_hub

# Clear existing data and reseed
rails db:reset

# Or just seed (without clearing)
rails db:seed
```

#### What Gets Created:

```

   Admin: admin@shophub.com / password

✅ Categories: 6 (with hierarchy)
   - Electronics (Phones, Laptops)
   - Clothing (Men's, Women's)

✅ Products: 10 active products

✅ Orders: 5-15 random orders

✅ Addresses: 2 per user (home + shipping)

✅ Attachments: Product images and PDFs

✅ Comments: Random comments on products and orders
```

Products are created with:

- **Description**: 3-sentence lorem ipsum

- **Stock**: Random between 0 and 100
- **SKU**: Auto-generated (e.g., `PROD-A1B2C3D4`)

### Order Statuses

The order tracking system supports these statuses:

```ruby

  pending: 0,           # Order created, awaiting payment
  payment_received: 1,  # Payment confirmed
  processing: 2,        # Order being prepared
  packed: 3,            # Order packed, ready to ship
  shipped: 4,           # Order shipped/out for delivery
  out_for_delivery: 5,  # Order is with delivery partner
  delivered: 6,         # Order delivered to customer
  ready_for_pickup: 7,  # Order ready for customer pickup
  picked_up: 8,         # Customer picked up order
  cancelled: 9          # Order cancelled
}
```

## Customizing Seed Data

Edit `db/seeds.rb`:


## Troubleshooting

### Issue: "Validation failed: Email has already been taken"

**Solution:** The seed is trying to create a user that already exists.

```bash
# Option 1: Reset the database
rails db:reset

# Option 2: Delete existing users first
rails console
User.destroy_all
```

### Issue: "No products found" when running order tracking seed

**Solution:** Run main seed first to create products.

```bash
rails db:seed
rails runner db/seeds_order_tracking.rb
```

### Issue: "Total amount must be greater than 0"

**Solution:** The order service is being used correctly in the tracking seed. This shouldn't happen with the provided seed file. If it does:

```bash
# Check that products exist and have prices
rails console
Product.pluck(:id, :name, :price)
```

### Issue: Seed takes too long

**Solution:** Reduce the number of iterations in `db/seeds.rb`:

```ruby
# Instead of 10.times
5.times do
  Product.create!(...)
end
```

## Best Practices

### 1. **Use db:reset for Clean Slate**

```bash
# This drops, creates, migrates, and seeds
rails db:reset
```

### 2. **Keep Seed Files Idempotent**

Use `find_or_create_by` for critical records:

```ruby
admin = User.find_or_create_by!(email: "admin@shophub.com") do |user|
  user.password = "password"
  user.first_name = "Admin"
  user.last_name = "User"
  user.role = :admin
end
```

### 3. **Use Separate Seed Files for Different Purposes**

- `db/seeds.rb` - Main application data
- `db/seeds_order_tracking.rb` - Order status testing
- `db/seeds_performance.rb` - Large dataset for performance testing
- `db/seeds_demo.rb` - Demo data for presentations

### 4. **Document Your Seeds**

Always add comments explaining what data is created and why:

```ruby
# Create admin user for accessing admin panel
admin = User.create!(...)

# Create products in Electronics category for testing search filters
electronics_products = 10.times.map do
  Product.create!(...)
end
```

### 5. **Use Faker for Realistic Data**

```ruby
# Better than hardcoded values
name: Faker::Commerce.product_name
email: Faker::Internet.email
description: Faker::Lorem.paragraph
```

## Data Reset Commands

```bash
# Full reset (drops DB, recreates, migrates, seeds)
rails db:reset

# Reset specific table
rails console
Product.destroy_all
Order.destroy_all

# Then reseed
rails db:seed

# Or use a transaction (rollback if error)
rails console
ActiveRecord::Base.transaction do
  User.destroy_all
  Product.destroy_all
  # ... create new data ...
end
```
