# Інтеграція з Новою Поштою

## Огляд

Повна інтеграція з API Нової Пошти v2.0 для автоматичного підвантаження міст, відділень та поштоматів під час оформлення замовлення.

## ✨ Реалізовані функції

### Для клієнтів

1. **Вибір типу доставки**
   - 🏪 **Відділення** - доставка у відділення Нової Пошти
   - 📦 **Поштомат** - доставка у поштомат (автоматизована комірка)

2. **Автокомпліт міста**
   - Введіть перші 2-3 літери назви міста
   - Автоматично з'являються підказки з API Нової Пошти
   - Показує місто та область
   - Наприклад: "Київ", "Львів", "Одеса", "Дніпро"

3. **Автоматичне завантаження відділень**
   - Після вибору міста автоматично завантажуються доступні точки
   - Відділення або поштомати (залежно від обраного типу)
   - Показує повну адресу кожної точки
   - Відображає номер відділення/поштомату

4. **Перевірка даних**
   - Обов'язкові поля: місто, відділення, номер телефону
   - Візуальна індикація обраних елементів
   - Помилки валідації на українській мові

### Для адміністраторів

1. **Перегляд замовлень**
   - Замовлення Нової Пошти позначені `delivery_method: "nova_poshta"`
   - Повна інформація про доставку у полі `delivery_address`
   - Формат: Тип, Місто, Адреса відділення, Телефон

2. **Обробка замовлень**
   - Стандартний процес обробки замовлень
   - Автоматичні email-повідомлення клієнтам
   - Можливість додавання трек-номеру Нової Пошти

## 🚀 Початок роботи

### Крок 1: Отримання API ключа (опціонально)

Для тестування можна працювати без ключа, але для production потрібен справжній ключ:

1. Відвідайте https://my.novaposhta.ua/settings/index#apikeys
2. Зареєструйтеся або увійдіть в особистий кабінет
3. Згенеруйте API ключ
4. Додайте його в файл `.env`:

```bash
NOVA_POSHTA_API_KEY=ваш_api_ключ_тут
```

### Крок 2: Запуск додатку

```bash
# Варіант 1: Використання Makefile
make start

# Варіант 2: Окремо запустити backend та frontend
# Terminal 1 (Backend):
cd /Users/vkuzm/RubymineProjects/shop_hub
rails server

# Terminal 2 (Frontend):
cd /Users/vkuzm/RubymineProjects/shop_hub/frontend
npm run dev
```

### Крок 3: Тестування

1. Відкрийте браузер: http://localhost:5175
2. Додайте товари в кошик
3. Перейдіть до оформлення замовлення
4. Оберіть "Nova Poshta" як спосіб доставки
5. Протестуйте:
   - Введіть "Київ" в поле міста
   - Оберіть місто з випадаючого списку
   - Перемкніться між "Відділення" та "Поштомат"
   - Оберіть відділення/поштомат
   - Введіть номер телефону: +380501234567
   - Завершіть замовлення

## 🔌 API Ендпоінти

### 1. Пошук міст

```
GET /api/v1/nova_poshta/cities?query=Київ
```

**Відповідь:**
```json
{
  "success": true,
  "data": [
    {
      "ref": "8d5a980d-391c-11dd-90d9-001a92567626",
      "name": "Київ",
      "name_ru": "Киев",
      "area": "Київська область",
      "settlement_type": "місто"
    }
  ]
}
```

### 2. Отримання відділень

```
GET /api/v1/nova_poshta/warehouses?city_ref=8d5a980d-391c-11dd-90d9-001a92567626
```

**Відповідь:**
```json
{
  "success": true,
  "data": [
    {
      "ref": "...",
      "number": "1",
      "description": "Відділення №1: вул. Хрещатик, 1",
      "short_address": "Хрещатик, 1",
      "latitude": "50.4501",
      "longitude": "30.5234"
    }
  ]
}
```

### 3. Отримання поштоматів

```
GET /api/v1/nova_poshta/postomats?city_ref=8d5a980d-391c-11dd-90d9-001a92567626
```

**Відповідь:**
```json
{
  "success": true,
  "data": [
    {
      "ref": "...",
      "number": "3001",
      "description": "Поштомат №3001: ТРЦ Ocean Plaza",
      "short_address": "ТРЦ Ocean Plaza"
    }
  ]
}
```

- Червоні повідомлення при незаповнених полях
- Текст на українській мові
- Чіткі інструкції що потрібно виправити

## 🔧 Технічні деталі

### API Клієнт (Ruby)

**Клас:** `NovaPoshta::ApiClient`

**Методи:**
- `search_cities(query)` - Пошук міст
- `get_warehouses(city_ref, warehouse_type:)` - Відділення
- `get_postomats(city_ref)` - Поштомати

**Обробка помилок:**
- Таймаути (10 секунд)
- Логування помилок
- Виключення `NovaPoshta::ApiClient::ApiError`

### Frontend (React)

**Хуки:**
- `useState` - стан форми, підказок, завантаження
- `useEffect` - автоматичне завантаження відділень
- `useCallback` - debounced пошук міст

**Автокомпліт:**
- Пошук при введенні 2+ символів
- Dropdown з'являється автоматично
- Закривається при виборі або blur

### Валідація

**Backend:**
```ruby
validates :delivery_address, presence: true, if: -> { nova_poshta? }
```

**Frontend:**
```typescript
if (!novaPoshtaData.selectedCity || 
    !novaPoshtaData.selectedWarehouse || 
    !novaPoshtaData.recipientPhone) {
  setError("Будь ласка, заповніть всі поля доставки Нової Пошти");
  return;
}
```

## 📝 Формат збереження адреси

Адреса зберігається в полі `delivery_address` у такому форматі:

```
Nova Poshta
Тип: Відділення (або Поштомат)
Місто: Київ
Відділення №1: вул. Хрещатик, 1
Телефон: +380501234567
```

Це дозволяє:
- Легко читати в адмін-панелі
- Друкувати на етикетках
- Показувати клієнту в підтвердженні замовлення

## 🐛 Відомі обмеження

1. **API ключ**
   - Без ключа API може не працювати або повертати обмежені дані
   - Отримайте безкоштовний ключ на https://my.novaposhta.ua

2. **Лімітування запитів**
   - Nova Poshta API має ліміти на кількість запитів
   - Наразі немає кешування відповідей
   - Для production рекомендується додати Redis кеш

3. **Мова інтерфейсу**
   - Основний інтерфейс англійською
   - Секція Нової Пошти на українській
   - Можна локалізувати весь додаток
